# Arquivo de Requisitos Arquivados - Milestone 4 (v4.0)

Este documento registra a validação e entrega dos requisitos de produto mapeados para a **Milestone 4 (Armazenamento e Entrega de Materiais)**, arquivada em 28 de Maio de 2026.

## Requisitos de Armazenamento e Entregáveis (FILE)
*   **[x] REQ-026**: Roteamento seguro e gravação de arquivos em diretório em disco local (`storage/` fora da pasta pública) no desenvolvimento local.
    *   *Resultado*: Criada classe `DiskStorageProvider` em `lib/storage.ts` salvando em `.storage/` oculto com metadados JSON `.meta`. Rotas de API protegidas NextAuth v5 servindo arquivos através de streams de Node.
*   **[x] REQ-027**: Máquina de estados de entregáveis arquitetônicos (`DRAFT`, `PENDING_REVIEW`, `APPROVED`, `APPROVED_WITH_CHANGES`, `REJECTED`, `DELIVERED`).
    *   *Resultado*: Lógica de transição de negócios implementada na Server Action `app/actions/deliverable.ts`, bloqueando edições físicas ou metadados de pranchas nos status finais `APPROVED` ou `DELIVERED`.
*   **[x] REQ-028**: Versionamento de arquivos de entregas e registro de feedback detalhado de revisão do cliente.
    *   *Resultado*: Server Action `submitNewVersion` realiza o increment automático de versão do arquivo no banco e preserva o histórico estruturado de revisões e feedbacks do cliente na coluna JSON `reviewComments`.
